About

就職活動のため、これまでの制作物をまとめて掲載するためのポートフォリオサイトを制作した。
採用担当をペルソナとして設定し、Figmaを用いてUI設計を行った。
ポートフォリオサイトも制作物の一部になると考え、UXや制作過程などを記すページを制作することにした。

User Experience

企業の新卒採用担当をペルソナとして想定した。
これまでの就職活動で出会った採用担当の方や社員の方々をモデルに作成している。

User Interface

rough

まずはiPadの「コンセプト」というアプリを使ってポートフォリオサイトのラフを描いた。
制作過程の中で、より使いやすくするために微調整した点はあるが、大まかにはこのラフ通りに制作している。

prototype

先ほど作成したラフを元に、Figmaでプロトタイプを作成した。
デスクトップでの環境を想定してmarginやpaddingなども考えながら作成したが、コーディング中にレスポンシブ対応での制作に切り替えたため不要になった設定が多かった。

できるだけシンプルで無駄のないデザインにし、コンテンツの中身が見やすいように心がけた。
他のページへ移動するためのメニューはコンパクトな実装が可能であるハンバーガーメニューを選んだ。
また、プロトタイプ・本サイト共に、font awesomeのアイコンを用いた。

Coding

プロトタイプではコーディングを意識しながらシンプルなデザインを想定していたため、コーディングの技術面で不可能なデザインはなく、比較的スムーズにコーディングをすることができたと感じた。
実際にコーディングに移ってからは、追加で必要だと考えられるものを付け足した。この要素については以下の通りである。

prompt to scroll

ページごとに関連するアイキャッチ画像をトップに大きく表示するというデザインであるため、ユーザーがスクロールをしなければならないことに気づかない場合が想定された。
そのため、トップの画像の下にスクロールを促す記号を表示させている。
この記号やトップの画像をクリックすることでも直下のコンテンツに自動スクロールできるようにした。
また、この実装に伴い記号をクリックしたときにゆっくりスクロールするようなアニメーションもjQueryで実装している。

slider

suishoのUIを表示させる際、全部で4画面あるUIの画像を全て並べると冗長になり比較しづらいと考えた。
そこで、スライダーを設置して横にスライドできるようにjQueryを用いて実装した。
ユーザーがスライダーに気づかないことが想定されたので、自動でスライドするよう実装し、現在何枚目かを示すドットを配置している。
また、スマートフォンでのスワイプにも対応している。

image

imgタグに対してドロップシャドウを適用し、画像であることを強調した。
当初はドロップシャドウを適用する予定はなかったが、画像の境界がわかりづらかったために追加で実装した。

▲ドロップシャドウがない場合
▲ドロップシャドウがある場合

また、拡大が必要な画像に対してはクリックしてモーダルを表示できるようにしている。
モーダル表示の際には背景を暗くし、バツ印のボタン以外にも背景をクリックしてモーダルを閉じられるようにした。
そのほかにも、比較したい2枚の画像はデスクトップ上で2つ横並びにしているが、スマートフォンで見る際には縦並びになるようにしたりと小さな工夫を凝らしている。

navigation

デスクトップで閲覧している場合には、ページの中での現在位置を表すナビゲーションを画面左側に設置した。
ユーザーが今何を見ているのか迷わないようにするだけでなく、見出しの役割も果たしている。
これによりページがあとどのくらいあるのかが分かりやすくなり、見たい内容にすぐ飛ぶことができる。
スマートフォンで閲覧している場合は画面の要素が増えすぎてしまうことを避けるため、非表示にしている。
