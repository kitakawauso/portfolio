Abstract

競技かるたの払いの動作を可視化するためのWebシステム。
「競技かるたの払いの動作の可視化」という題で大学4年次から卒業研究として制作開始。

競技者の競技能力の向上が目的である。
競技かるたでは、各選手の体格に合った構えを習得したり、払いを熟達させることが必須である。
そのほかにも上達するために必要な要素はあるが、構えや払いは人の目で判断するよりも測定した数値の方が正確だろうという仮定から、構えや払いを測定することを考えた。

機械学習を用いて姿勢推定を行い、得られた情報を可視化することで選手の取りについて分析を行う。
複数人の情報を用いて比較を行うことで、より詳細な分析ができる。

recoding system

札を払っている様子を撮影し、 姿勢推定をするためのシステムの画面。
機械学習による姿勢推定は、特別な撮影用機械を用いなくとも分析を行えることが強みである。
Mediapipeを用いており、指先も含めた33箇所の部位の座標を細かく測定できる。

visualization system

前述のシステムで得られた情報を可視化するシステムの画面。
Javascriptを用いて描画している。三次元ボーン表示を行うことで、 通常は見られない任意の角度で払いの様子を観察できる。
グラフ表示でも比較をすることで、より詳細な分析ができる。

User Experience

user persona

競技かるたを始めたばかりの選手をこのシステムのペルソナとして設定した。
身長が低い選手は構えや払いを習得するのが難しく、測定による分析の必要性が高いと考えている。

customer journey map

このシステムを用いることで小柄な上級者はペルソナと比べて前傾姿勢で構えていることなどがわかり、その情報をもとに自身に合う構えを見つけていく、という流れを想定した。

User Interface

現状では操作方法が分かりづらく、また今後可視化手法の追加を検討しているため、システムのUIを一新する予定である。
未実装ではあるが、考案中の設計図を掲載する。
現状でのシステムについてはこちら

選手の情報の読み込み画面をレコーディング画面とは別の画面にし、 ユーザーが操作に迷うことなくシステムの利用ができるように改善した。
レコーディング画面では右側に全ての操作パネルを設置し、 上から順に操作することでレコーディングが完了する。

▲1人用の可視化システム
▲複数人比較用の可視化システム

可視化手法の追加に伴い情報量が多くなることを見込んで、 可視化画面は一人用の可視化と複数人で比較用の可視化の2画面に分けた。
ボーン表示とグラフ表示はマウスによる操作で自在に視点を変えることができ、 直感的な操作で可視化を分析することができる。
